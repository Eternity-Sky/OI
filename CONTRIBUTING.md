# 贡献指南

## 开发流程

1. Fork 项目
2. 创建功能分支
3. 提交更改
4. 发起 Pull Request

## 代码规范

- 使用 2 空格缩进
- 遵循 ESLint 规则
- 提交信息遵循 Conventional Commits

## 分支命名

- 功能分支：feature/功能名
- 修复分支：fix/问题描述
- 优化分支：optimize/优化内容

## 提交信息格式

- feat: 新功能
- fix: 修复问题
- docs: 文档更新
- style: 代码格式
- refactor: 代码重构
- test: 测试相关
- chore: 构建过程或辅助工具的变动
